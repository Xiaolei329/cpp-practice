面向对象：
遇到问题就抽象，把问题抽象成具备某些属性和方法的某类的
某个对象去执行对应的方法。

封装：把一些属性或方法变成类内部可见，信息隐藏；
对外提供一些接口来进行访问； 更安全。

继承：子类与父类之间关系，可以把公共的属性和方法 放到
父类里面，多个子类继承父类的属性和方法，可以减少代码重复
提高复用性；

多态：不同的对象对同一个信息产生不同的行为。
举个例子：
比如 猫，狗，鸡 都可以继承 动物这个类；
但是他们三个在执行叫()这个动作时，却表现了不同的表现；

类的定义：
 class Person{
    public:
	  void study();   ----成员函数
	  void exam();
	private:
	  string name;    ----成员变量
	  int age;
 };
 
 //类中声明，类外实现
 //在类外面实现 成员函数
 void Person::study(){
 
 }
 
 友元函数，友元类不受这个限制；
 类的成员函数不受这个限制；
 
 public：可被外部创建的对象直接调用；
        可被继承的类的对象调用；
 
 protected：不可以被外部创建的对象调用；
    但是可以被公有继承的类的对象调用；

 private：不能被外部创建的对象调用；
    也不能被继承的类的对象调用；  
 
注意：
当没有使用任何的访问控制关键字时，默认算是private！


创建对象：
Person per1; ---创建对象 栈中

Person *per2 = new Person();---堆中
用完需要delete(per2);

两种方式下的对象调用成员函数：
 per1.study();
 per2->study();

【封装】：
对对象的属性信息隐藏在对象内部；
通过类提供的函数接口来实现对属性的访问和设置值；

好处：
1.更安全，对象的属性值，不可以随意的获取和设置了；
2.访问时：可以增加控制；
  设置时：可以进行合法性检查；


【this指针】：
this指针是一直存在的，某个对象调用成员函数时候，
因为成员函数是公用的，会默认传递过去一个this指针；
this指针指向的是当前的对象。
所以，在成员函数中，可以直接使用this指针。
  只能在成员函数中使用;
  

1）静态函数是没有this指针的，静态函数如同静态变量一样，不属于具体的哪一个对象
2）只有成员函数才有this指针，友元函数不是类的成员函数，没有this指针  



构造函数：

创建对象时候，编译器自动调用的；
构造函数与类名相同；
构造函数是用来给对象初始化的；
没有写构造函数时，会自动生产一个无参构造函数；
但是一旦定义了构造函数，就不再自动生成无参的构造函数；
构造函数不需要返回值类型，不能return；


演示一下：
构造函数的重载；
构造函数的列表初始化；
含有成员对象的类，初始化时：先执行成员对象的构造函数，
再执行类的构造函数；
