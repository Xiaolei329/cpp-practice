
一、引用
就是为变量取一个别名，对这个别名的操作就等于对变量的操作；


1.基本变量的引用：
int a = 100;

如果是指针：
运行操作系统的位数32；
编译器位数32；

int *p;----4字节 8字节
p = &a;
这个时候对*p的操作就等价于 对a的操作；


C++里面的引用：
int &r = a;//定义引用时候必须直接初始化；

这个时候对 r的操作就等价于 对a的操作；


2.怎么区别&什么时候是引用，什么时候是取地址？？
 如果是变量定义的地方出现&,(包括参数列表里面) 就是引用；
 如果是在其他地方出现&,就是指针里面的取地址；
 
 
3.引用有什么用？ 
最最常用地方是函数参数列表里面用引用。
表示会直接操作的是实参原来的值；

C里面参数传递两种方式：？
1.传值方式；
2.传地址方式；

C++: 
fun(int &a,  int& b){
  xxxxx
}

fun(c, d);

举代码例子说明；
引用方式传递参数；

4.引用 与 指针的区别？
相同点
1）参数传递时，用引用和用指针都可以实现对原来的实参的更改；

int *p;----4字节
p = &a;

int a;
a = 100;


不同点
1）引用必须是某个变量的别名，不存在空引用；
  定义时候必须同时初始化；

2）引用只是变量的第二个标签别名，不开辟内存；

3）指针可以自增，自减，偏移等操作，
  引用不可以。

扩展知识：
::几种出现的地方：
::是作用域运算符，A::B表示作用域A中的B，A可以是名字空间、类；

1.全局作用域符号
当全局变量在局部函数中与其中某个变量重名，
那么就可以用::来区分；
　　char zhou; //全局变量 
　　void sleep() 
　　{ 
	　　char zhou; //局部变量 
	　　zhou(局部变量) = zhou(局部变量) *zhou(局部变量) ; 
	　　::zhou(全局变量) = ::zhou(全局变量) *zhou(局部变量); 
　　}
2.类作用; ::前面放类名，表示取的是这个类下面的成员
为了避免不同的类有名称相同的成员而采用作用域的方式进行区分
例如：
    A,B表示两个类，在A,B中都有成员member。那么
　　A::member就表示类A中的成员member
　　B::member就表示类B中的成员member

三、new 和 delete

1.开辟的位置是在堆区；返回是首地址；
  如果细分：
  new 开辟是在自由存储区； 用delete释放；
  malloc开辟是在堆区，用free释放
  C++编译器默认使用堆来实现自由存储；有一定差别；
  
2.开辟基本变量时：
int *p = new int; //没有初始化
int *p = new int(10);//有初始化
deletd p; ---释放普通变量空间

3.开辟数组时：
int *p = new int[10];
deletd[]p; ---释放数组空间

4.new 一个对象；
    Student stu = Student();  //栈中分配 
    Student* stu = new Student();  //堆中分配 
    delete stu; 
	
1）new创建类对象需要指针接收,一处初始化，多处使用
new创建类对象使用完需delete销毁;
2）new创建对象直接使用堆空间，而局部不用new定义类对象
则使用栈空间;
3）Student stu = Student();这种方式创建的对象，
内存分配到栈里；使用 “.” 调用对象的方法
stu.fun();

Student* stu = new Student(); 这种方式创建的对象，
内存分配到堆里；使用“->” 调用对象的方法
stu->fun();
在堆中的对象不会自动删除，内存不会自动回收，
所以new一个对象使用完毕，必须调用delete，释放内存空间

5.new 和 malloc 区别：
1.不用详细指定开辟空间的字节数，
malloc申请内存空间时，手动计算所需大小,
new只需类型名，自动计算大小;

2.malloc申请的内存空间不会初始化，new可以初始化

3.malloc的返回值为void*,接收时必须强转，new不需要;


扩展知识1：
内存的分配方式有三种
1）从静态存储区域分配。
内存在程序编译的时候就已经分配好，这块内存在程序的整个运行期间都存在。
例如全局变量，static 变量。 
2）在栈上创建。在执行函数时，
函数内局部变量，函数参数
在栈上分配内存空间效率很高，但是分配的内存容量有限。
3）从堆上分配的。
malloc 或 new 申请任意多少的内存，
程序员自己负责在何时用 free 或 delete 释放内存


扩展知识2：
Java 中的 new 和 C++ 中的 new 的区别
java中new返回对象引用(别名)，而c++中返回的是对象的实际地址(指针)。
java中没有delete的原因是java有垃圾回收机制，
当一个对象没有被引用时，系统会自动将其清理掉(也就是系统自动执行了delete)。
